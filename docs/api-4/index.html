模式4 - 纯接口模式+JSON返回

请求方式#

POST请求，Content-Type为application/json

请求URL#

/payway/order/mode4

请求字段#

字段类型长度示例可否为空说明是否签名
apiKey字符串-"tc"否商务后台创建的apiKey(亲联系您的上级代理,获取该标识,以及对应的签名key)是
thirdNo字符串16-32"12345678"否商户单号(16-32位,任意字符)是
amount字符串-"100"否交易金额(单位:元),不能小于等于0是
customerPayType字符串-"ZFB"否支付宝：ZFB是
customerAccountNo字符串-"622848****5314"是非必传，收付款时的账户是(传则需签名)
customerAccountName字符串-"张三"是当receiveType为"UNIONPAY"是,该字段必传,且与银行卡开户人姓名相同是(必传需签名)
customerId字符串32"abc001"是不同用户ID必须不同，同一个用户ID必须相同（32位任意字符）该字段传则签名，不传则不签名
notifyUrl字符串-"1"否异步通知url,该url请不要携带特殊符号,请正常填写如"http://****/order/notify"是
orderType字符串32"buy"是buy/sellbuy(入金)/sell(出金)
timestamp字符串-"1675454774574"是时间戳传则需要签名，不传则不签名
sign字符串-"1"否签名否

注: customerPayType 改用户的收付款方式可通过接口获取收付款方式列表

返回内容#

字段类型示例说明
code整型200请求后台返回操作状态，200为成功操作
msg字符串"OK"对code进行说明
data实体对象请参考返回示例｜
sign字符串----签名

data返回内容#

字段类型示例说明
thirdNo字符串"1245648978477"商户单号
orderNo字符串"998154152121156156"otc订单号
createTime字符串"1678734859487"订单创建时间
finishTime字符串"1678734859487"订单完结时间
timestamp字符串"1678734859487"时间戳
amount字符串"100"下单金额
quantity字符串"14.6842878"资产数量
customerPayType字符串"ZFB"支付宝(ZFB)
customerAccountNo字符串"6228480234564561654"用户银行卡/支付宝/微信账号
customerAccountName字符串"张三"用户银行卡账户姓名
status字符串"成功"成功,失败,待确认,初始状态,下单中
dealerPayType字符串---币商收/付款方式
dealerAccountNo字符串---币商收付款账户
dealerAccountName字符串---｜币商收付款账户昵称
dealerBankName字符串---币商银行卡名称
dealerNickname字符串---币商挂单昵称
type字符串“buy”订单的买卖方向buy/sell
mgs字符串“”订单状态说明
